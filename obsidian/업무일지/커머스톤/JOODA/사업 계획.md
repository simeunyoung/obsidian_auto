
### AI가전 개발 데이터 확보 및 공동활용 지원

홈페이지 : EC2

데이터 검색 및 필터링

메타데이터 저장 : Aurora DB

대용량 학습 데이터 저장 : S3

다운로드 : S3 Pre-signed URLs

데이터 업로드 및 전처리 : Lambda or batch

사용자 인증 및 권한 관리 :  Amazon Cognito / AWS IAM

####  데이터를 구매해서 대용량의 데이터를 어떻게 업로드하는게 제일 효율적인가?
1. **AWS CLI (Command Line Interface) 또는 AWS SDK:**
    
    - **가장 일반적이고 유연한 방법입니다.** aws s3 cp 명령어나 aws s3 sync 명령어를 사용하면 됩니다.
        
    - **멀티파트 업로드(Multipart Upload):** AWS CLI 및 SDK는 대용량 파일을 자동으로 작은 조각(part)으로 나누어 병렬로 업로드합니다. 이 방식은 네트워크 오류 시 재시작이 쉽고, 대용량 파일 업로드 시간을 크게 단축시킵니다. GB에서 TB 단위 파일에 매우 효율적입니다.
        
    - **aws s3 sync:** 로컬 디렉토리와 S3 버킷을 동기화하여 변경된 파일만 업로드하거나 삭제된 파일을 반영할 때 유용합니다.
        
    - **장점:** 스크립트화 용이, 자동 멀티파트 업로드, 높은 신뢰성.
        
    - **단점:** 직접 관리해야 하는 서버(EC2 또는 온프레미스)가 필요.
        
2. **AWS Transfer Family:**
    
    - **기능:** SFTP, FTPS, FTP 프로토콜을 사용하여 S3 버킷으로 파일을 직접 업로드하고 다운로드할 수 있는 완전 관리형 서비스입니다.
        
    - **장점:** 기존 SFTP/FTP 기반 시스템과 쉽게 통합 가능, 서버 관리 불필요, 대용량 파일 전송에 최적화.
        
    - **단점:** CLI/SDK보다 비용이 더 발생할 수 있음.
        
3. **AWS Direct Connect 또는 AWS VPN:**
    
    - **기능:** 온프레미스 데이터센터와 AWS 간에 전용 네트워크 연결(Direct Connect) 또는 암호화된 터널(VPN)을 설정하여 안정적이고 빠른 대용량 데이터 전송을 가능하게 합니다.
        
    - **장점:** 매우 안정적인 고대역폭 연결, 예측 가능한 성능.
        
    - **단점:** 설정 비용 및 시간이 많이 소요.
        
4. **AWS Snow Family (매우 대용량 - 페타바이트급):**
    
    - **기능:** 물리적인 스토리지 디바이스(Snowball, Snowball Edge, Snowmobile)를 AWS에서 보내주면, 데이터를 로컬에서 디바이스에 복사한 후 다시 AWS로 보내 S3에 업로드하는 서비스입니다. 네트워크 전송이 비효율적일 정도의 초대용량(수십 TB ~ 페타바이트) 데이터에 적합합니다.
        
    - **장점:** 네트워크 대역폭 제약 없이 초대용량 데이터 전송 가능.
        
    - **단점:** 물리적 배송 시간 소요, 대용량 데이터에만 효율적.
        

**효율성 추천:** 대부분의 GB~TB 단위 데이터 업로드에는 **AWS CLI 또는 SDK를 사용한 멀티파트 업로드**가 가장 효율적이고 비용 효과적인 방법입니다.
