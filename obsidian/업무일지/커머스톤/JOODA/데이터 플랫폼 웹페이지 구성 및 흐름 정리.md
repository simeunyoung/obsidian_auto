

**1. 랜딩 페이지 (Landing Page)**

- **목적:** 잠재 사용자에게 플랫폼의 가치와 특징을 소개하고, 로그인/회원가입으로 유도하는 첫 접점.
    
- **주요 내용:**
    
    - 플랫폼의 핵심 가치 제안 (예: "고품질 학습용 대용량 데이터 제공").
        
    - 주요 이점 및 특징 (예: "검증된 기업 데이터", "빠른 다운로드").
        
    - 플랫폼 이용 대상 명시 (예: "승인된 기업 고객 전용").
        
    - 플랫폼 사용 예시 또는 성공 사례 (선택 사항).
        
    - 명확한 **"로그인" / "회원가입" 버튼** 또는 콜투액션(CTA).
        
    - 문의하기 링크 (Q&A 또는 직접 문의).
        

**2. 로그인 페이지 (Login Page)**

- **목적:** 기존 사용자가 계정 정보를 입력하여 플랫폼에 접근.
    
- **주요 기능:**
    
    - 사용자 ID (또는 이메일) 및 비밀번호 입력 폼.
        
    - **"회원가입" 페이지로 이동하는 링크.**
        
    - **"ID/PW 찾기" 페이지로 이동하는 링크.**
        
    - 로그인 버튼.
        
    - 로그인 상태 유지 (Remember Me) 옵션 (선택 사항).
        

**3. 회원가입 페이지 (Registration Page)**

- **목적:** 새로운 사용자가 플랫폼 계정을 생성.
    
- **주요 기능:**
    
    - 사용자 정보 입력 (이름, 회사 이메일 주소, 비밀번호).
        
    - **기업 정보 입력 (최초 기업 등록 시):** 기업명, 사업자 등록 번호, 주소, 연락처, 사업자 등록증 사본 업로드 등
        
    - 이메일 인증 절차.
        
    - 이용 약관 및 개인정보처리방침 동의.
        
    - 회원가입 버튼.
        

**4. ID/PW 찾기 페이지 (Forgot ID/Password Page)**

- **목적:** 사용자가 ID를 찾거나 비밀번호를 재설정.
    
- **주요 기능:**
    
    - **ID 찾기:** 가입 시 사용한 이메일/전화번호로 ID 전송.
        
    - **비밀번호 재설정:** 이메일/SMS 인증 후 새로운 비밀번호 설정.
        

**5. 메인 페이지 (Main Page)**

- **목적:** 로그인 후 사용자에게 플랫폼의 핵심 기능과 주요 정보에 대한 개요를 제공.
    
- **주요 기능:**
    
    - 환영 메시지 (사용자 이름, 소속 기업명).
        
    - **주요 섹션으로의 빠른 링크:** "데이터 검색", "마이페이지", "업로드" 등.
        
    - **추천/인기 데이터셋:** 사용자의 관심사나 최신 동향에 맞는 데이터셋 추천.
        
    - 최근 업데이트된 데이터셋 목록.
        
    - 공지사항 또는 중요 알림.
        

**6. 검색 페이지 (Search/Explore Page)**

- **목적:** 사용자가 필요한 데이터셋을 효과적으로 찾아볼 수 있도록 지원.
    
- **주요 기능:**
    
    - 강력한 검색 바 (키워드, 고급 검색 옵션).
        
    - 다양한 필터링 옵션 (데이터 유형, 카테고리, 태그, 생성/업데이트 날짜, 데이터 크기, 제공 기업 등).
        
    - 검색 결과 목록 (데이터셋 이름, 요약 설명, 메타데이터 일부).
        
    - 정렬 옵션 (관련성, 최신순, 크기순 등).
        
    - 페이지네이션 또는 무한 스크롤.
        
    - 검색 결과에서 **"상세 페이지"로 이동하는 링크**.
        

**7. 마이 페이지 (My Page)**

- **목적:** 사용자 개인 정보 및 활동 내역 관리.
    
- **주요 기능:**
    
    - **개인 정보 수정:** 이름, 비밀번호, 연락처 등.
        
    - **나의 기업 정보:** 소속 기업 정보 조회 (기업명, 등록일, 접근 가능한 데이터셋 목록).
        
    - **다운로드 요청 내역:** Large On-Demand Download 요청 상태(진행 중, 완료, 실패) 및 완료된 요청의 다운로드 링크.
        
    - **업로드 내역:** 사용자가 업로드한 데이터셋 목록.
        
    - 알림 설정.
        
    - API 키 관리 (선택 사항, 고급 사용자용).
        

**8. Q&A 페이지 (Q&A / Support Page)**

- **목적:** 사용자의 질문에 답하고 지원을 제공.
    
- **주요 기능:**
    
    - **자주 묻는 질문(FAQ):** 일반적인 질문과 답변 모음.
        
    - **문의하기 폼:** 관리자에게 직접 질문을 보낼 수 있는 양식.
        
    - 문의 내역 조회 (내가 과거에 보낸 문의 목록 및 답변).
        
    - 이용 가이드/튜토리얼 링크 (선택 사항).
        

**9. 관리자 페이지 (Admin Page)**

- **목적:** 플랫폼 운영 및 관리 (일반 사용자에게는 접근 불가).
    
- **주요 기능:**
    
    - **기업 관리:** 기업 목록 조회, 승인/거부, 정보 수정, 접근 권한 설정 (어떤 기업이 어떤 데이터셋을 다운로드할 수 있는지).
        
    - **사용자 관리:** 사용자 목록 조회, 권한 변경, 계정 활성/비활성화, 삭제.
        
    - **데이터셋 관리:** 모든 데이터셋 조회, 승인/비승인 (업로드된 데이터셋 검토), 메타데이터 수정, 삭제, 버전 관리.
        
    - **업로드/다운로드 모니터링:** 전체 트래픽, 사용량, 에러 로그 모니터링.
        
    - 시스템 설정 (카테고리, 태그, 알림 템플릿 등).
        
    - 보고서 및 통계.
        

**10. 업로드 페이지 (Upload Page)**

- **목적:** 승인된 기업의 사용자가 데이터를 플랫폼에 업로드.
    
- **주요 기능:**
    
    - 파일/폴더 선택 기능 (드래그 앤 드롭 지원).
        
    - 업로드할 데이터셋에 대한 메타데이터 입력 폼 (이름, 설명, 유형, 태그, 라이선스, 접근 권한 등).
        
    - 업로드 진행 상태 표시 (진행률, 남은 시간).
        
    - 업로드 완료/실패 알림.
        
    - 업로드 가이드라인 제공.
        

**11. 상세 페이지 (Detail Page)**

- **목적:** 특정 데이터셋에 대한 모든 상세 정보를 제공하고, 다운로드 옵션을 제시.
    
- **주요 기능:**
    
    - 데이터셋 이름, 버전, 상세 설명.
        
    - 모든 메타데이터 (파일 형식, 크기, 스키마, 생성/수정일, 제공 기업 등).
        
    - 데이터 미리보기/샘플 (이미지, 표, 그래프 등).
        
    - **다운로드 옵션:**
        
        - **Small On-Demand Download 버튼:** 작은 파일을 즉시 다운로드 (Pre-signed URL 생성).
            
        - **Large On-Demand Download 요청 버튼:** 대용량 파일 다운로드 요청 (배치 작업 트리거) 및 요청 완료 시 알림 수신 동의.
            
    - 관련 데이터셋 추천.
        
    - 데이터셋 이용 가이드.
