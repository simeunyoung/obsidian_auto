# 🚀 SEO 최적화 홍보 페이지 프로젝트 기술 사양서

## 1. 프로젝트 개요

- **목적:** 검색 엔진 최적화(SEO)를 극대화하여 유기적 유입을 늘리고, 빠른 로딩 속도로 전환율을 높이는 홍보 페이지 구축
    
- **핵심 목표:** Lighthouse 성능 점수 90점 이상, 구글/네이버 1페이지 노출 기반 마련
    

## 2. 기술 스택 (Tech Stack) 및 선택 이유

|**분류**|**기술**|**선택 이유**|
|---|---|---|
|**Framework**|**Astro**|**Zero-JS:** 기본적으로 자바스크립트가 없는 정적 HTML을 생성하여 SEO 및 LCP(로딩속도) 최적화에 최강|
|**Language**|**TypeScript**|코드 안정성 확보 및 팀원 간 타입 공유를 통한 협업 효율 증대|
|**Deployment**|**Vercel**|Edge Network 기반의 빠른 서빙, GitHub 연동을 통한 자동 배포(CI/CD) 최적화|
|**Domain**|**가비아**|국내 도메인 관리 편의성 (Vercel 네임서버 연결 예정)|
|**Analytics**|**GA4 + GSC**|방문자 행동 분석(GA4) 및 구글 검색 노출 현황 모니터링(Google Search Console)|
|**Styling**|**Tailwind CSS**|유틸리티 퍼스트 방식으로 일관된 UI 개발 및 빌드 시 미사용 스타일 자동 제거(최경량화)|

## 3. 주요 SEO 구현 전략 (팀 공유용)

### A. 개발 파트 (Technical SEO)

- **Content Collections:** 블로그 포스팅이나 공지사항은 Astro의 내장 기능을 사용하여 마크다운(.md) 기반으로 관리.
    
- **Image 최적화:** 모든 이미지는 Astro의 `<Image />` 컴포넌트를 사용하여 WebP 변환 및 Lazy-loading 강제 적용.
    
- **Sitemap/Robots:** `@astrojs/sitemap` 패키지를 사용하여 빌드 시마다 최신 사이트맵 자동 생성.
    
- **SSR vs SSG:** 대부분의 페이지는 SSG(정적 생성)로 빌드하여 속도를 극대화하되, 실시간 데이터가 필요한 경우에만 부분적 하이드레이션(Islands Architecture) 사용.
    

### B. 기획/마케팅 파트 (Content SEO)

- **Meta 데이터 관리:** 페이지별 `Title`, `Description`, `OG Tag` 리스트 작성 및 전달.
    
- **키워드 전략:** 타겟 키워드를 도출하고 이를 본문(H1, H2 태그)에 자연스럽게 녹여내는 작업.
    
- **구조화 데이터:** FAQ나 서비스 정보를 JSON-LD 스키마 마크업으로 정리하여 개발팀에 전달.
    

## 4. 협업 워크플로우

1. **Git Flow:** `main` 브랜치는 운영, `develop` 브랜치에서 기능 개발. Vercel Preview 기능을 통해 PR 단계에서 SEO 결과물 미리 확인.
    
2. **도메인 설정:** 가비아에서 구매한 도메인의 네임서버를 Vercel 주소로 변경하여 SSL 인증서 자동 갱신 및 연결.
    
3. **검색엔진 등록:** 사이트 런칭 즉시 Google Search Console 및 네이버 서치어드바이저에 사이트맵 제출.
    

## 5. 향후 확장 계획

- **백링크 작업:** 보도자료 배포 및 커뮤니티 바이럴을 통한 외부 권위 점수 확보.
    
- **성능 모니터링:** Vercel Speed Insights를 통해 실제 사용자 환경에서의 성능 지표 상시 모니터링.
