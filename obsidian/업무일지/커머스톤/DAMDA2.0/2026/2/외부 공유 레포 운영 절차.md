
### 설정 정보

- 기존 레포 : STSC
- 기존 브랜치(공유할) : release
- 공유 레포 : client-share
- 공유 브랜치 : stsc

### 최초 1회: 기존 레포에 공유 remote 추가

```
기존 레포에서
git remote -v
git remote add client https://github.com/DAMDA-KEA/client-share.git
git remote -v
```

### 매번 공유: release 최신 -> 공유 레포에 스냅샷 반영(히스토리 없이)
- 공유 레포 히스토리 노출 방지 + .env.qa만 포함하는 방식

```
** POWERSHELL로 진행 **

# 기존 레포 최신화
/STSC
git checkout release
git pull origin release

# 스냅샷 생성
/client-share
git checkout --orphan stsc-clean
git rm -r --cached . 2>$null
Get-ChildItem -Force | Remove-Item -Recurse -Force

# 최신 코드 복사 + env 정책 적용
robocopy ..\STSC . /E /XD .git .idea /XF .env.dev .env.prod .env.local

# 커밋 + 원격 덮어쓰기
git add .
git commit -m "stsc share snapshot (release update)"
git push -f origin stsc-clean:stsc

# 임시 브랜치 삭제
git branch -D stsc-clean

```
