
**문서 유형:** 서비스 운영 절차서 (SOP) 
**서비스명:** 담다 (IoT 플랫폼) **버전:** v1.1

---

## 1단계. 서비스 적합성 확인

고객사의 제품이 담다 서비스 이용 가능한지 사전 확인한다.

- 데이터 분석(KEA) 및 IoT 지원 가능 여부 검토
- 담다 앱 사용 여부 확인
    - **사용 가능:** 담다 앱 설치 후 바로 사용
    - **자체 앱 개발 필요:** 템플릿 앱 제공

**협의 항목**

|구분|항목|
|---|---|
|필수|서비스 가능 여부, APP|
|선택|OTA, BLE(툭), Open API, 시각화, 스마트싱스|

---

## 2단계. 데이터셋 작성

포탈 등록을 위한 데이터셋을 준비한다.

- 데이터셋 초안 작성 후 고객사에 발송, 작성 요청
- 고객사 작성 완료 후 내용 검토
- 누락 항목 확인 및 추가 작성 요청
- 위 검토 → 요청 과정을 데이터셋이 완성될 때까지 반복
- 데이터 보고 방식 결정
- 최종 완료 후 포탈에 데이터 등록
    - 유형, 모델, 리소스
    - 액션, 트리거
    - APP 화면 등록

---

## 3단계. ESP32 통신 모듈 전달

IoT 통신 모듈을 빌드하여 고객사에 전달한다.

- 고객사가 ESP32 모듈을 직접 구매하여 당사로 발송
- 수령한 모듈에 코드 빌드 후 고객사로 반송

**함께 제공하는 자료**

| 항목  | 내용                                                                                    |
| --- | ------------------------------------------------------------------------------------- |
| 문서  | 필수 : 디바이스 개발 가이드, 플랫폼 to 디바이스, 플랫폼 to 앱, 앱 to 디바이스<br>템플릿 앱 사용 시 :템플릿 앱 운영 가이드, 기능정의서 |
| 소스  | device code                                                                           |
| APP | app code                                                                              |

---

## 4단계. ESP32 및 앱 테스트 (더미 데이터)

더미 데이터를 활용해 통신 모듈 및 앱 동작을 검증한다.

**테스트 대상 시스템**

| 구분  | 항목                                  |
| --- | ----------------------------------- |
| 필수  | Device, APP, DCM, DSIM, IoT Core    |
| 선택  | EM, MM, FPN, OAuth, ST(SmartThings) |

**테스트 방법**

- 기능 정의서 제공 후 항목별 TC 제공 및 테스트 진행
- 테스트 모니터링 및 지원 병행

**테스트 항목**

| 구분  | 항목                                       |
| --- | ---------------------------------------- |
| 필수  | 디바이스 등록, 삭제                              |
| 선택  | OTA, 그룹 관리, 회원 관리, SmartThings 연동, OAuth |

---

## 5단계. 수요기업 서버 최종 테스트 (실데이터)

4단계와 동일한 TC(테스트 케이스)를 사용하되, 실제 데이터로 검증한다.

- 4번 테스트와 동일한 TC 사용
- 더미 데이터 → 실데이터(MCU)로 교체하여 재검증
- 데이터 정합성 및 서비스 동작 최종 확인

---

## 6단계. 운영 최종 테스트

테스트 환경에서 운영 환경으로 전환한다.

- 운영으로 타입 변경 후 제공 및 안내
    - APP 설정 운영 전환
    - Device 설정 운영 전환
    - Portal 데이터 운영 등록
- 5번 테스트와 동일한 TC로 운영 환경 검증

---

## 7단계. CS (고객 지원)

운영 이후 고객사 CS 지원 체계로 전환하여 운영한다.
