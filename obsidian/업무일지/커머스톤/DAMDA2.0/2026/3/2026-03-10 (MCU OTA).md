
### 펌웨어 업데이트 구조 설계(AWS)

#### 1. 통신 모듈
 - FreeRTOS OTA 업데이트 작업 생성
```
# 시퀀스

1. 운영자가 FreeRTOS OTA 작업 생성  
2. DB에 iot_model_firmware 생성  
3. DB에 iot_device_firmware = QUEUED  
4. ESP32가 start-next  
5. ESP32가 S3에서 자기 firmware 다운로드  
6. ESP32 self update  
7. ESP32가 SUCCEEDED/FAILED 보고  
8. Rule → Lambda  
9. DB update_status 갱신
```

#### 2. MCU
- 사용자 지정 작업 생성

MCU OTA는 보통 gateway(통신 모듈)가 대신 업데이트 한다.
```
Cloud
 ↓
ESP32 (gateway)
 ↓
MCU
```


- MCU firmware를 S3에서 직접 다운로드
```
IoT Core
 ↓
ESP32
 ↓
S3 firmware download
 ↓
MCU flash


# json document 예시

{  
"operation": "mcu_ota",  
"version": "2.3.1",  
"artifactUrl": "https://s3/.../mcu_fw.bin",  
"checksum": "sha256:xxxxx"  
}
```

```
# 시퀀스

1. 운영자가 Custom Job 생성
2. job document 안에 MCU firmware S3 URL 포함
3. DB에 iot_model_firmware 생성
4. DB에 iot_device_firmware = QUEUED
5. ESP32가 start-next
6. ESP32가 job document 수신
7. ESP32가 S3에서 MCU firmware 다운로드
8. ESP32가 MCU bootloader 통해 펌웨어 전달
9. MCU flash / verify
10. ESP32가 SUCCEEDED/FAILED 보고
11. Rule → Lambda
12. DB update_status 갱신
```
