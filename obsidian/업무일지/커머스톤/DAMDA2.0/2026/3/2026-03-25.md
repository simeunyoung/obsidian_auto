

# OTA Check API — MCU/ESP32 구분 응답으로 변경

> [!info] 변경 목적 OTA 업데이트 여부를 단순 `boolean` 대신 **MCU / ESP32 각각의 업데이트 상태**를 구분하여 모바일 앱에 노출

---

## API 변경

### Before

```http
POST /api/1.0/device/ota/check
```

```json
true
```

### After

```http
POST /api/1.0/device/ota/check
```

```json
{
  "mcu": true,
  "esp32": false
}
```

---

## 변경 파일

### `IotModelFirmware.java`

- `job_type` 컬럼 매핑 필드 추가
    - `String jobType`

### `IotDeviceFirmwareRepository.java`

| 구분  | 내용                                                                                |
| --- | --------------------------------------------------------------------------------- |
| 기존  | `findLatestInProgressFirmwareByDeviceId()` — QUEUED 상태 펌웨어 단건 조회                  |
| 변경  | `existsQueuedByDeviceIdAndJobType(deviceId, jobType)` — job_type별 QUEUED 존재 여부 조회 |

### `OtaCheckResponseDto.java` ⭐ 신규

```java
boolean mcu;
boolean esp32;
```

### `IotDeviceService.java`

- `otaCheck()` 반환 타입 변경
    - `boolean` → `OtaCheckResponseDto`

### `IotDeviceServiceImpl.java`

- MCU, ESP32 각각 `existsQueuedByDeviceIdAndJobType()` 호출 후 응답 DTO 구성

### `OtaController.java`

- 반환 타입 변경
    - `boolean` → `OtaCheckResponseDto`
