

#### IoT Rule 네이밍 규칙
```
iot-{domain}-{source}-{purpose}[-to-{target}]
```

| 항목          | 의미      | 예                       |
| ----------- | ------- | ----------------------- |
| iot         | 고정      | iot                     |
| domain      | 기능 영역   | jobs, device, session   |
| source      | 이벤트 출처  | mqtt, events, presence  |
| purpose     | 무엇을 함   | status-sync, trace, log |
| to-{target} | 목적지(옵션) | to-db, to-lambda        |
