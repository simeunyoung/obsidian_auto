
# 성능 테스트 방법


### 1. Device > IoT Core > DSim (MQTT 게시/구독) 모니터링

이 구간에서는 주로 **Amazon CloudWatch Logs**와 **CloudWatch Metrics**를 사용합니다.

#### 1.1 IoT Core 모니터링

**1.1.1 CloudWatch Logs 설정 (IoT Core 작업 로깅)**

IoT Core는 CloudWatch Logs와 통합되어 디바이스 연결/연결 해제, 메시지 게시/구독 시도 등 다양한 작업에 대한 로깅을 제공합니다.

- **설정 방법:**
    
    1. AWS IoT Core 콘솔로 이동합니다.
        
    2. 왼쪽 탐색 창에서 "Settings"를 클릭합니다.
        
    3. "Logs" 섹션에서 "Edit"을 클릭합니다.
        
    4. "Log level"을 "Info" 또는 "Debug"로 설정합니다 (성능 테스트 시에는 상세한 로그를 위해 "Debug"가 유용할 수 있지만, 비용 및 볼륨을 고려해야 합니다).
        
    5. "CloudWatch log group"을 선택하거나 새로 생성합니다. 예를 들어, /aws/iot/logs 와 같은 이름으로 생성할 수 있습니다.
        
    6. "IAM role"을 생성하거나 선택하여 IoT Core가 CloudWatch Logs에 로그를 푸시할 수 있는 권한을 부여합니다.
        
    7. "Save"를 클릭합니다.
        
- **확인 방법:**
    
    1. CloudWatch 콘솔로 이동합니다.
        
    2. 왼쪽 탐색 창에서 "Log groups"를 클릭합니다.
        
    3. 설정한 IoT Core 로그 그룹 (예: /aws/iot/logs)을 선택합니다.
        
    4. 여기서 디바이스의 연결/연결 해제, MQTT 메시지 게시/구독 시도에 대한 로그를 확인할 수 있습니다. 메시지 ID, 타임스탬프 등을 통해 개별 메시지의 흐름을 추적할 수 있습니다.
        

**1.1.2 CloudWatch Metrics (IoT Core 지표)**

IoT Core는 CloudWatch에 자동으로 다양한 지표를 게시합니다.

- **확인 방법:**
    
    1. CloudWatch 콘솔로 이동합니다.
        
    2. 왼쪽 탐색 창에서 "Metrics"를 클릭합니다.
        
    3. "AWS/IoT" 네임스페이스를 선택합니다.
        
    4. 여기서 다음과 같은 지표들을 확인할 수 있습니다:
        
        - MessagesPublishedIn (게시된 메시지 수)
            
        - MessagesDeliveredOut (구독자에게 전달된 메시지 수)
            
        - ConnectionAttempts (연결 시도 횟수)
            
        - ConnectSuccess (연결 성공 횟수)
            
        - PublishInRate (초당 게시율)
            
        - PublishOutRate (초당 전달율)
            
        - Latency (메시지 전달 지연 시간) - 이 지표는 메시지가 IoT Core에 도착하여 구독자에게 전달되기까지의 시간을 나타냅니다.
            
    
    이 지표들을 통해 MQTT 메시지 처리량과 기본적인 지연 시간을 파악할 수 있습니다.
    

#### 1.2 DSim 애플리케이션 모니터링

DSim은 사용자가 직접 개발한 모듈이므로, 이 모듈 내부에서 로그와 지표를 수집하도록 코드를 추가해야 합니다.

- **로그 (CloudWatch Logs):**
    
    - **설정 방법:** DSim 애플리케이션이 실행되는 환경(예: Lambda, EC2)에서 CloudWatch Logs로 로그를 푸시하도록 설정합니다.
        
        - **Lambda:** Lambda 함수는 기본적으로 CloudWatch Logs에 로그를 푸시합니다. DSim 코드를 Lambda로 작성했다면, console.log (Node.js) 또는 print() (Python) 등을 사용하여 로그를 남기면 자동으로 CloudWatch Logs에서 확인할 수 있습니다.
            
        - **EC2:** EC2 인스턴스에서 실행되는 경우, CloudWatch Agent를 설치하여 DSim 애플리케이션의 로그 파일을 CloudWatch Logs로 스트리밍하도록 설정합니다.
            
    - **DSim 코드 내 로깅:**
        
        - MQTT 메시지를 구독한 시간
            
        - DB 삽입을 시작한 시간
            
        - DB 삽입이 완료된 시간
            
        - 각 메시지 처리 시 걸린 총 시간
            
        - 처리된 메시지 수
            
        - 오류 발생 시 상세 정보
            
- **지표 (CloudWatch Metrics - Custom Metrics):**
    
    - **설정 방법:** DSim 코드 내에서 PutMetricData API 호출을 통해 사용자 정의 지표를 CloudWatch에 게시합니다.
        
    - **DSim 코드 내 지표 생성 예시:**
        
        - 구독된 MQTT 메시지 수 (DSim/MessagesSubscribed)
            
        - RDS 삽입 성공 횟수 (DSim/RDSInsertsSuccessful)
            
        - 개별 메시지 처리 시간 (DSim/MessageProcessingLatency)
            
        - 개별 RDS 삽입 시간 (DSim/RDSInsertLatency)
            
- **확인 방법:**
    
    1. CloudWatch 콘솔로 이동합니다.
        
    2. "Log groups"에서 DSim 애플리케이션의 로그를 확인하여 상세한 처리 과정을 분석합니다.
        
    3. "Metrics"에서 DSim 네임스페이스 아래의 사용자 정의 지표들을 확인하여 처리량과 지연 시간을 시각화합니다.
        

#### 1.3 End-to-End 지연 시간 측정

디바이스에서 IoT Core를 거쳐 DSim까지의 전체 지연 시간을 측정하려면, 메시지 내에 타임스탬프를 포함시키는 것이 가장 효과적입니다.

- **방법:**
    
    1. **디바이스:** MQTT 메시지를 게시하기 직전에 현재 타임스탬프를 메시지 페이로드에 포함시킵니다 (예: JSON 형식의 { "data": "...", "timestamp": <현재 Unix 타임스탬프> }).
        
    2. **DSim:** DSim에서 이 메시지를 구독하여 받은 후, 페이로드의 타임스탬프와 현재 시간을 비교하여 전체 지연 시간을 계산합니다. 이 지연 시간을 CloudWatch Custom Metric으로 게시하거나 로그에 기록합니다.
        

### 2. RDS (DSim > DB 적재) 모니터링

RDS의 성능 모니터링은 **Amazon CloudWatch**와 **Amazon RDS Performance Insights**를 주로 사용합니다.

#### 2.1 CloudWatch Metrics (RDS 기본 지표)

RDS는 CloudWatch에 자동으로 다양한 지표를 게시합니다.

- **확인 방법:**
    
    1. CloudWatch 콘솔로 이동합니다.
        
    2. 왼쪽 탐색 창에서 "Metrics"를 클릭합니다.
        
    3. "RDS" 네임스페이스를 선택합니다.
        
    4. 여기서 다음과 같은 지표들을 확인할 수 있습니다:
        
        - CPUUtilization (CPU 사용률)
            
        - DatabaseConnections (DB 연결 수)
            
        - FreeStorageSpace (여유 스토리지 공간)
            
        - ReadIOPS, WriteIOPS (읽기/쓰기 IOPS)
            
        - ReadThroughput, WriteThroughput (읽기/쓰기 처리량)
            
        - CommitLatency, DMLLatency (커밋 및 DML 지연 시간)
            
        - NetworkReceiveThroughput, NetworkTransmitThroughput (네트워크 수신/송신 처리량)
            
    
    이 지표들을 통해 RDS 인스턴스의 전반적인 건강 상태와 DSim의 부하가 DB에 미치는 영향을 파악할 수 있습니다.
    

#### 2.2 RDS Performance Insights

Performance Insights는 데이터베이스 로드를 시각적으로 분석하여 문제를 신속하게 감지하는 데 도움을 줍니다.

- **설정 방법:**
    
    1. RDS 콘솔로 이동합니다.
        
    2. 왼쪽 탐색 창에서 "Performance Insights"를 클릭합니다.
        
    3. 모니터링할 DB 인스턴스를 선택합니다. Performance Insights가 활성화되어 있지 않다면, DB 인스턴스 수정 시 "Performance Insights" 섹션에서 활성화할 수 있습니다.
        
- **확인 방법:**
    
    1. Performance Insights 대시보드에서 "DB load" 그래프를 확인합니다. 여기서 DSim이 DB에 가하는 로드 패턴을 시각적으로 파악할 수 있습니다.
        
    2. "Top SQL", "Top Hosts", "Top Users" 등을 통해 어떤 SQL 쿼리가 가장 많은 시간을 소모하는지, DSim 인스턴스가 얼마나 많은 부하를 주는지 등을 분석할 수 있습니다. 특히 DSim에서 실행하는 INSERT 쿼리의 성능을 여기서 상세히 분석할 수 있습니다.
        
    3. 각 INSERT 쿼리당 걸리는 시간과 총 적재량은 Performance Insights의 Top SQL 탭에서 특정 쿼리의 실행 통계를 통해 간접적으로 파악하거나, DSim 애플리케이션에서 직접 로깅/지표로 수집하는 것이 가장 정확합니다.
        

### 3. 통합 모니터링 및 대시보드

이러한 모든 로그와 지표를 한눈에 볼 수 있도록 **CloudWatch Dashboards**를 구성하는 것이 좋습니다.

- **설정 방법:**
    
    1. CloudWatch 콘솔로 이동합니다.
        
    2. 왼쪽 탐색 창에서 "Dashboards"를 클릭합니다.
        
    3. "Create dashboard"를 클릭하고 이름을 지정합니다.
        
    4. 다양한 위젯(그래프, 숫자 통계, 로그 쿼리 등)을 추가하여 앞서 설명한 모든 지표와 로그 그룹의 중요한 정보를 한 화면에 표시합니다.
        
        - IoT Core의 메시지 처리량
            
        - DSim의 구독 및 적재 성공률, 지연 시간 (Custom Metrics)
            
        - RDS의 CPU, IOPS, DB Load (Performance Insights 그래프도 연동 가능)
            
        - DSim 애플리케이션의 오류 로그를 필터링하는 CloudWatch Logs Insight 쿼리 결과
            
- **확인 방법:** 대시보드를 열어 시스템의 실시간 성능과 건강 상태를 모니터링합니다.
    

### 요약 및 권장 사항

1. **IoT Core:**
    
    - **로그:** CloudWatch Logs로 IoT Core 작업 로깅 설정 (Debug 레벨 고려).
        
    - **지표:** CloudWatch Metrics AWS/IoT 네임스페이스에서 MessagesPublishedIn, MessagesDeliveredOut, Latency 등 확인.
        
2. **DSim (애플리케이션):**
    
    - **로그:** CloudWatch Logs로 DSim 애플리케이션 로그 푸시 (Lambda는 자동, EC2는 CloudWatch Agent).
        
        - 로그 내용: 메시지 수신 타임스탬프, DB 삽입 시작/완료 타임스탬프, 처리 시간, 오류.
            
    - **지표:** CloudWatch Custom Metrics (PutMetricData)를 사용하여 MessagesSubscribed, RDSInsertsSuccessful, MessageProcessingLatency, RDSInsertLatency 등 게시.
        
3. **RDS:**
    
    - **지표:** CloudWatch Metrics RDS 네임스페이스에서 CPUUtilization, DatabaseConnections, WriteIOPS, WriteThroughput 등 확인.
        
    - **상세 분석:** RDS Performance Insights를 활성화하여 "DB Load", "Top SQL" 등을 통해 INSERT 쿼리 성능 분석.
        
4. **End-to-End 지연 시간:** MQTT 메시지 페이로드에 타임스탬프를 포함하여 DSim에서 최종 지연 시간 계산 및 Custom Metric으로 게시.
    
5. **통합 시각화:** CloudWatch Dashboards를 사용하여 모든 핵심 지표와 로그를 한눈에 모니터링.
    

이러한 서비스들을 조합하여 설정하면, 디바이스에서 IoT Core를 거쳐 DSim, 그리고 최종적으로 RDS에 데이터가 적재되는 전 과정의 성능을 상세하게 모니터링하고 분석할 수 있을 것입니다.
