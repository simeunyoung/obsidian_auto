AWS 성능테스트
	1. Publish-In, Publish-out 시간 구하기
	2. 지표 대시보드 확인
	3. IoTCore에서 퍼블리시 아웃 될 수 있는 모든 작업 비활성화

### 지표 설명
### 1. **PublishIn.Success**

- 디바이스 → IoT Core 로 MQTT 메시지를 성공적으로 퍼블리시한 횟수
    
- 즉, **클라이언트가 IoT Core에 메시지를 보낸 성공 카운트**
    

### 2. **PublishOut.Success**

- IoT Core → 구독자(디바이스/서버) 로 MQTT 메시지를 성공적으로 전달한 횟수
    
- 즉, **IoT Core가 메시지를 브로커에서 나가도록 성공 처리한 카운트**
    

### 3. **Subscribe.Success**

- 클라이언트가 `SUBSCRIBE` 요청을 보냈을 때 IoT Core가 이를 성공적으로 처리한 횟수
    
- 특정 토픽 구독 성공 횟수
    

### 4. **Connect.Success**

- MQTT 클라이언트가 IoT Core에 연결(`CONNECT`)을 시도해 성공한 횟수
    
- 세션 연결 성공 수치
    

### 5. **Ping.Success**

- MQTT `PINGREQ` / `PINGRESP` 를 통한 keep-alive 성공 횟수
    
- 즉, 클라이언트와 IoT Core 사이의 연결이 정상적으로 유지된 횟수
