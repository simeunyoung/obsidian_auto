- AWS IoT OTA(Over-the-Air)
	- DEVICE에 새로운 펌웨어를 안전하게 전송하는 기능

1. cert_config.txt 파일 준비(인감증명서 신청서 작성)
```
[ req ]  
prompt = no  
distinguished_name = my_dn  
x509_extensions = my_exts

[ my_dn ]  
commonName = admin@godamda.kr
organizationName = KEA  
organizationalUnitName = DAMDA  
countryName = KR  
stateOrProvinceName = Seoul  
localityName = Mapo-Gu

[ my_exts ]  
keyUsage = digitalSignature
extendedKeyUsage = codeSigning
```
	[ req ] : 기본적인 규칙을 정의, prompt = no는 국가, 도시 등을 일일이 물어보지 말고, 아래 [my_dn] 섹션에 정의된 값을 자동으로 사용하라는 의미
	[ my_dn ] : 인증서 소유자의 신원 정보
	[ my_exts ] :  인증서의 '용도'와 '규칙'을 정의하는 확장(Extensions) 섹션
	keyUsage = digitalSignature : '디지털 서명'에 사용할 수 있다는 기본적인 용도를 명시
	extendedKeyUsage = codeSigning : 더욱 구체적으로 '소프트웨어 코드(펌웨어)에 서명하는 용도'로만 사용될 것임을 명시

2. 개인 키(private key) 생성(디지털 인감 만들기)
```
openssl genpkey -algorithm EC -pkeyopt ec_paramgen_curve:P-256 -pkeyopt ec_param_enc:named_curve -outform PEM -out ecdsasigner.key
```

3. 인증서 생성(인감증명서 발급 받기)
	- 개인 키(ecdsasigner.key)가 진짜 내 것이 맞다고 공개적으로 증명해주는 인증서를 생성
	- 공개되어도 안전
```
openssl req -new -x509 -config cert_config.txt -extensions my_exts -nodes -days 365 -key ecdsasigner.key -out ecdsasigner.crt
```
