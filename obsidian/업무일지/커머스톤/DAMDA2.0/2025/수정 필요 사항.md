

### OAUTH2.0
	OAuth2.0 JWT 사용중
	JWT privateKey는 서버를 시작 할 때 마다 자동으로 생성해서 사용중
	ALB의 Sticky Session 기능 사용중
	 -> 서버 시작하면 jwt 비밀키 생성 (이중화 되어있다면 두 서버는 다른 키를 사용) 되고,
	 이중화 된 서버들이 서로 다른 키를 사용하기 때문에 인증이 안되는게 맞으나 sticky session
	 을 사용해 A서버를 이용했다면 A서버로만 가기 때문에 문제 없이 사용중인 상황
	 하지만 만약 서버를 무중단 배포 하려고 서버 1, 2번을 순서대로 재시작한다면 1번 서버를 사용
	 중이던 사용자 모두 로그아웃, 2번 서버를 사용중이던 사용자 모두 로그아웃 되기 때문에 구시대
	 적인 방식으로 배포해야함

	해결 방법:
	KeyStore(.jks)파일을 써서 키를 고정시켜야함
```
// src/main/resources로 이동해서 아래 명령어 입력

keytool -genkeypair -alias damdakey -keyalg RSA -keysize 2048 -storetype JKS -keystore damda-key.jks -validity 3650 -storepass changeit

// 기존의 jwkSource 메서드를 이걸로 교체
    @Bean
    public JWKSource<SecurityContext> jwkSource() {
        try {
            // 1. resources 폴더에서 .jks 파일을 찾음(yml에서 불러오는 걸로 변경)
            String path = "my-key.jks"; 
            String alias = "mykey";      // 아까 명령어 칠 때 쓴 alias
            String password = "changeit"; // 아까 명령어 칠 때 쓴 비밀번호

            // 2. 파일 읽기
            KeyStore keyStore = KeyStore.getInstance("JKS");
            ClassPathResource resource = new ClassPathResource(path);
            
            try (InputStream inputStream = resource.getInputStream()) {
                keyStore.load(inputStream, password.toCharArray());
            }

            // 3. 파일 안에 있는 RSA 키 꺼내기
            RSAKey rsaKey = RSAKey.load(keyStore, alias, password.toCharArray());

            // 4. JWKSet으로 만들기 (이제 서버 재시작해도 키가 똑같음!)
            JWKSet jwkSet = new JWKSet(rsaKey);
            return new ImmutableJWKSet<>(jwkSet);

        } catch (Exception e) {
            log.error("JKS 파일 로드 실패! : ", e);
            throw new IllegalStateException("키스토어를 불러올 수 없습니다.", e);
        }
    }
```
