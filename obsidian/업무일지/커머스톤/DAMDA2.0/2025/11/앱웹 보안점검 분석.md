## 앱/웹 보안 점검 분석 및 보완 방법 점검

1. 패스워드 정책(심은영,정민영)
	1. 정책 강화
		대문자, 소문자, 숫자, 특수문자 중 3가지 이상을 조합하여 8자리 이상 길이로 구성
2. IDOR 취약점(임동혁)
	1. 웹 프록시 툴로 userIdx를 변경하여 관리자 기능 악용
		1. Request Param 대신 Session 사용
		2. ROLE 관리
3. 악성 스크립트 XSS 취약점(임동혁)
	1. 치환
4. 인증 우회(심은영,정민영)
	1. Activity 직접 실행
		==Activity는 MainActivity만 존재해 강제 실행해 기능 악용 되지 않음==
	2. API 호출 시 토큰/세션 검증안됨
		- login3으로 로그인해서 login4 정보 요청해도 정보를 주는 취약점
		1. JWT 검증 필요
5. 과도한 권한 설정(정민영)
	1. AndroidManifest.xml 내 userCleartextTraffic=false로 설정 변경
6. 민감 정보 노출(정민영, 심은영)
	1. 1차 개인정보 마스킹제공
	2. 2차 인증 후 개인정보 제공
	3. Firebase API Key 폐기 후 재발급 및 설정
7. 디바이스-앱 통신 시 중요 정보 암호화 미흡(정민영)
	1. HTTPS통신
	2. 암/복호화
8. 루팅/탈옥 기기 정상 작동(정민영)
	1. 루팅 기기 탐지 기능 추가(라이브러리)
