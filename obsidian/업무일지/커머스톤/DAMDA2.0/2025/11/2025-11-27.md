Git
- **목표**: `KEA_Portal-master` 및 `KEA_DAMDA_WEB-master` 프로젝트를 GitHub 원격 저장소에 Push.
- **문제**: 프로젝트 내 **바이너리 파일**(`aws-java-sdk-1.11.902.jar` (176MB), `hyper/hyperd` (159MB) 등)의 크기가 **GitHub 파일 크기 제한(100MB)**을 초과하여 `[remote rejected] (pre-receive hook declined)` 에러 발생.
- **해결 접근**: 대용량 파일 관리를 위해 **Git Large File Storage (Git LFS)**를 도입하고 히스토리를 정리함.
1.  Git LFS 설치
```
git lfs install
```
2. 대용량 파일 추적 설정
```
git lfs track "WebContent/WEB-INF/lib/aws-java-sdk-1.11.902.jar"
git lfs track "WebContent/WEB-INF/lib/hyper/hyperd"
```
3. LFS 포인터로 변경 및 커밋
```
git add .gitattributes
git add .
git commit -m "Configure Git LFS for large binary files"
```

**Unrelated Histories 강제 병합**: 로컬의 새 기록과 원격의 기존 기록(README 등)을 강제로 병합하여 두 히스토리를 연결함.
```
git pull origin main --allow-unrelated-histories
```
