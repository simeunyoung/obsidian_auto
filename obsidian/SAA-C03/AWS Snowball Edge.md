- 대량의 데이터를 빠르게 AWS로 전송하거나, 온프레미스에서 엣지 컴퓨팅을 수행할 수 있는 물리적인 디바이스
- 인터넷을 통해 데이터를 전송하는 대신 **실제 물리적인 장치(Snowball Edge)** 를 AWS에서 배송해 주고, 사용자가 데이터를 전송한 후 AWS로 다시 보내는 방식
---
## **🔹 주요 특징**

### **1️⃣ 대량 데이터 전송 (데이터 마이그레이션)**

- **TB~PB(페타바이트) 규모의 데이터를 AWS로 전송할 때 사용**해.
    
- 인터넷 대역폭을 많이 차지하지 않고, **빠르고 안전하게 AWS로 데이터를 이동 가능**해.
    
- **AWS로 데이터를 전송하는 속도가 빠름** (인터넷 업로드보다 훨씬 빠름).
    

### **2️⃣ 엣지 컴퓨팅 (로컬에서 데이터 처리)**

- 단순한 데이터 이동뿐만 아니라, **엣지 컴퓨팅을 지원**해서 **AWS 서비스를 Snowball Edge에서 직접 실행 가능**해.
    
- 로컬에서 **데이터를 전처리하고 필터링한 후, 필요한 데이터만 AWS로 전송 가능**해.
    

### **3️⃣ 보안이 강력함**

- **256-bit AES 암호화 적용**
    
- **전송 중 데이터 보호를 위한 TPM(Trusted Platform Module) 적용**
    
- AWS에서 데이터 복사 후, **Snowball Edge 내부 데이터를 자동으로 삭제하여 보안 유지**

---
## **🔹 Snowball Edge를 사용하는 과정**

1️⃣ **AWS 콘솔에서 Snowball Edge 주문**  
2️⃣ **AWS에서 Snowball Edge 장치를 배송**  
3️⃣ **온프레미스에서 데이터를 Snowball Edge에 복사**  
4️⃣ **AWS로 Snowball Edge 장치를 다시 보냄**  
5️⃣ **AWS가 데이터를 Amazon S3로 업로드**  
6️⃣ **AWS에서 Snowball Edge 내부 데이터를 자동 삭제**

---

## **🔹 언제 Snowball Edge를 사용해야 할까?**

✔ **인터넷 업로드 속도가 느릴 때** (네트워크로 데이터 전송이 어려운 경우)  
✔ **TB~PB 규모의 데이터를 빠르게 AWS로 마이그레이션해야 할 때**  
✔ **엣지 컴퓨팅을 수행해야 할 때 (예: 네트워크 연결이 어려운 환경에서 AWS 서비스 실행)**  
✔ **온프레미스에서 데이터를 처리하고 필요한 데이터만 AWS로 전송할 때**
