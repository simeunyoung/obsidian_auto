## Amazon S3 스토리지 클래스 비교

| 스토리지 클래스                          | 특징                         | 비용(상대적)      | 사용 사례                              | 복원력/가용성          |
| --------------------------------- | -------------------------- | ------------ | ---------------------------------- | ---------------- |
| **S3 Standard**                   | 자주 액세스                     | 💰💰💰 (비쌈)  | 웹 콘텐츠, 모바일 앱, 데이터 분석               | ✅ 다중 AZ, 고가용성    |
| **S3 Intelligent-Tiering**        | 액세스 패턴 자동 분석 후 적절한 계층으로 이동 | 💰💰 (효율적)   | 액세스 패턴이 예측 불가한 데이터 (ex. 미디어, 로그 등) | ✅ 다중 AZ          |
| **S3 Standard-IA**                | 드물게 액세스, 하지만 빠른 접근 필요      | 💰 (저렴)      | 백업, 재해 복구 데이터, 오래된 로그              | ✅ 다중 AZ          |
| **S3 One Zone-IA**                | 드물게 액세스, 비용 절감 극대화         | 💰 (가장 저렴)   | 복원 가능하고 중복 필요 없는 데이터               | ❌ 단일 AZ (내구성 낮음) |
| **S3 Glacier Instant Retrieval**  | 자주 접근은 아니지만 몇 분 이내에 접근 필요  | 💰 (저렴)      | 오래된 이미지, 비디오, 기록                   | ✅ 다중 AZ          |
| **S3 Glacier Flexible Retrieval** | 몇 분~수시간 안에 접근 가능           | 💰 (더 저렴)    | 콜드 데이터, 규제 준수 목적 보관                | ✅ 다중 AZ          |
| **S3 Glacier Deep Archive**       | 거의 접근 안 함, 12시간 이내 복원      | 💰💰 (가장 저렴) | 법적 보관 데이터, 장기 백업                   | ✅ 다중 AZ          |
