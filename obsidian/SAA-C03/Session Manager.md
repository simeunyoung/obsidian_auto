**AWS Systems Manager**라는 서비스 안에 포함된 기능 중 하나로
👉 **EC2 인스턴스에 접속할 수 있게 해주는 웹 기반 터미널**이라고 생각하면 됨.

---
## 뭐가 그렇게 좋은데?

### 1. **SSH 키 필요 없음**

기존에는 `.pem` 키 파일 필요했음
Session Manager는 **키 없어도 EC2에 접속 가능**

### 2. **보안성↑**

- SSH 포트(22번)를 **열 필요 없음** → 공격 받을 구멍 줄어듦
    
- 접속 로그도 남아서 **감사 추적 가능**
    
- IAM으로 **접근 권한 제어 가능**
    

### 3. **브라우저에서 바로 접속**

AWS 콘솔 들어가서 **EC2 > 인스턴스 > ‘세션 시작’** 누르면  
바로 터미널 창 열림! (별도 프로그램 설치도 필요 없음)

---
## 사용 조건은?

1. EC2 인스턴스에 **SSM 에이전트**가 설치되어 있어야 함 (Amazon Linux는 기본 탑재)
    
2. EC2 인스턴스에 **IAM 역할** 연결돼 있어야 함
    
    - `AmazonSSMManagedInstanceCore` 정책 필요
        
3. 인스턴스가 **Systems Manager와 통신 가능해야 함**
    
    - 즉, 인터넷 or VPC 엔드포인트 통해 **SSM에 접근 가능해야 함**
    
---
## 실제 접속 흐름 요약

1. EC2 인스턴스에 IAM 역할 연결 (SSM 에이전트 + 정책 포함)
    
2. 콘솔에서 EC2 인스턴스 선택
    
3. "세션 시작" 버튼 클릭
    
4. 터미널 창에서 작업
