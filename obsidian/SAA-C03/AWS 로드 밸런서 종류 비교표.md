## AWS 로드 밸런서 종류 비교표

|항목|Application Load Balancer (ALB)|Network Load Balancer (NLB)|Gateway Load Balancer (GWLB)|
|---|---|---|---|
|**주 용도**|HTTP/HTTPS (L7) 트래픽 처리|TCP/UDP (L4) 고성능 처리|패킷 검사, 보안 어플라이언스 연결|
|**OSI 계층**|Layer 7 (애플리케이션 계층)|Layer 4 (전송 계층)|Layer 3/4 (네트워크/전송 계층)|
|**트래픽 처리**|URL, 헤더, 쿠키 기반 라우팅|IP, 포트 기반 라우팅|IP 패킷을 보안 어플라이언스로 포워딩|
|**로드밸런싱 대상**|EC2, Lambda, IP 주소, 컨테이너|EC2, IP 주소, 컨테이너|보안 장비 (가상 방화벽 등)|
|**특징**|- HTTP/HTTPS 전용  <br>- 고급 라우팅 규칙  <br>- WAF 사용 가능|- 낮은 지연 시간  <br>- 수십만 동시 연결 처리 가능  <br>- TLS 종료 가능|- 보안 트래픽 처리  <br>- 트래픽 검사와 필터링  <br>- 보안 벤더와 통합|
|**예시 사용**|웹 서버, API 서버|게임 서버, 실시간 채팅|방화벽, IDS/IPS, DPI 등 보안 장비|
|**대상 그룹 구성**|포트/경로 기준 라우팅|포트 기준|IP 기반|
|**Health Check**|HTTP 기반|TCP 기반 (또는 사용자 정의)|ICMP, TCP 등|
|**사용 가능 프로토콜**|HTTP, HTTPS|TCP, UDP, TLS|모든 IP 트래픽|

## 🧠 언제 어떤 걸 써야 할까?

### ✅ ALB

- 웹 애플리케이션, REST API
    
- URL 경로별로 다른 서비스로 라우팅하고 싶을 때
    
- WAF(Web Application Firewall)를 함께 쓰고 싶을 때
    

**예)**

- `/api` → API 서버
    
- `/img` → 이미지 서버
    

---

### ✅ NLB

- 초당 수십만 개의 요청 처리
    
- 실시간 통신, 게임 서버, VoIP 등 빠른 반응이 중요한 서비스
    
- TCP/UDP 레벨에서 로드 밸런싱 필요할 때
    

**예)**

- 채팅 앱, 메신저 백엔드
    
- IoT 디바이스와 통신 서버
    

---

### ✅ GWLB

- 가상 방화벽, 침입 탐지(IDS), 침입 방지(IPS) 같이 트래픽을 **분석/검사**해야 할 때
    
- 외부 트래픽을 어플라이언스로 **자동 전달**해서 필터링
    

**예)**

- 기업 보안 트래픽 분석
    
- 제로트러스트 보안 구조
    

---

## 🔚 정리 한 줄 요약

|상황|선택|
|---|---|
|웹 페이지, REST API 라우팅|✅ ALB|
|빠른 TCP/UDP 처리, 게임/실시간|✅ NLB|
|보안 검사, 방화벽 연동|✅ GWLB|
