### Well-Architected Framework 5대 원칙

| 원칙                              | 핵심 내용              | 관련 서비스                                   |
| ------------------------------- | ------------------ | ---------------------------------------- |
| 운영 우수성 (Operational Excellence) | 모니터링, 자동화, 지속적 개선  | CloudWatch, CloudTrail, AWS Config       |
| 보안 (Security)                   | 데이터 보호, 접근 제어, 암호화 | IAM, KMS, WAF, Shield                    |
| 신뢰성 (Reliability)               | 장애 대응, 복구 계획       | Auto Scaling, Route 53, S3, RDS Multi-AZ |
| 성능 효율성 (Performance Efficiency) | 적절한 리소스 선택, 확장성    | Lambda, EC2 인스턴스 타입, ElastiCache         |
| 비용 최적화 (Cost Optimization)      | 비용 절감, 온디맨드 활용     | AWS Budgets, Spot Instances, S3 Glacier  |

### **1️⃣ 운영 우수성 (Operational Excellence)**

> **시스템을 모니터링하고 개선하는 방법을 최적화하는 원칙**

### 📌 **핵심 개념**

- **운영 프로세스 자동화** (배포 자동화, 모니터링)
    
- **문제 발생 시 신속 대응** (CloudWatch, CloudTrail 활용)
    
- **지속적인 개선을 통한 효율성 증대**
    

### 🛠️ **관련 AWS 서비스**

- **AWS CloudWatch** – 리소스 및 애플리케이션 모니터링
    
- **AWS CloudTrail** – API 호출 로깅 및 감시
    
- **AWS Config** – 변경 사항 추적 및 규정 준수 확인
    

---

### **2️⃣ 보안 (Security)**

> **데이터 및 시스템을 보호하는 원칙**

### 📌 **핵심 개념**

- **IAM을 활용한 접근 제어**
    
- **데이터 암호화 및 보호**
    
- **보안 이벤트 모니터링 및 대응**
    

### 🛠️ **관련 AWS 서비스**

- **AWS IAM** – 사용자 및 권한 관리
    
- **AWS KMS** – 데이터 암호화 키 관리
    
- **AWS WAF & AWS Shield** – DDoS 및 웹 애플리케이션 보안
    
- **AWS GuardDuty** – 위협 감지
    

---

### **3️⃣ 신뢰성 (Reliability)**

> **시스템이 장애 발생 시 복구 가능하고, 요구사항 변화에 따라 적응할 수 있도록 설계하는 원칙**

### 📌 **핵심 개념**

- **시스템 장애 대비 (Fault Tolerance)**
    
- **자동 복구 및 확장 지원**
    
- **백업 및 복구 전략 필수**
    

### 🛠️ **관련 AWS 서비스**

- **Amazon Route 53** – DNS 장애 조치 (Failover)
    
- **AWS Auto Scaling** – 자동 확장 및 복구
    
- **Amazon S3 & RDS Multi-AZ** – 내구성 및 가용성 보장
    
- **AWS Backup** – 데이터 백업 및 복구 관리
    

---

### **4️⃣ 성능 효율성 (Performance Efficiency)**

> **비즈니스 요구사항을 충족하면서 최적의 성능을 제공하는 원칙**

### 📌 **핵심 개념**

- **적절한 리소스 선택 (EC2, Lambda 등)**
    
- **워크로드 특성에 맞게 최적화**
    
- **자동 확장을 통한 성능 최적화**
    

### 🛠️ **관련 AWS 서비스**

- **Amazon EC2 인스턴스 패밀리** – 워크로드에 맞는 최적의 인스턴스 선택
    
- **AWS Lambda** – 서버리스 컴퓨팅 활용
    
- **Amazon ElastiCache** – 데이터 캐싱으로 성능 향상
    
- **Amazon CloudFront** – 콘텐츠 전송 속도 최적화
    

---

### **5️⃣ 비용 최적화 (Cost Optimization)**

> **필요한 리소스를 최적 비용으로 사용하도록 설계하는 원칙**

### 📌 **핵심 개념**

- **불필요한 리소스 제거 및 자동화**
    
- **적절한 요금제 선택 (On-Demand, Spot, Reserved)**
    
- **스토리지 및 데이터 전송 비용 최적화**
    

### 🛠️ **관련 AWS 서비스**

- **AWS Cost Explorer & AWS Budgets** – 비용 모니터링 및 예산 관리
    
- **Amazon S3 Lifecycle** – 스토리지 비용 절감 (Glacier 활용)
    
- **EC2 Reserved & Spot Instances** – 비용 절감 옵션 활용
    
- **AWS Compute Optimizer** – 리소스 최적화 추천
## AWS 서비스별 실습 (실제 적용)

✅ **CloudWatch 알람 & CloudTrail 로그 분석** (운영 우수성)  
✅ **IAM 사용자/그룹/역할 만들고 권한 제어** (보안)  
✅ **RDS Multi-AZ 배포 & Route 53 장애 조치 실습** (신뢰성)  
✅ **Auto Scaling + ALB 연동** (성능 효율성)  
✅ **EC2 Reserved vs Spot 비교 + S3 라이프사이클 정책** (비용 최적화)
