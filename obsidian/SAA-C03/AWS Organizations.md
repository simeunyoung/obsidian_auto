### **📌 AWS Organizations의 주요 개념**

1️⃣ **조직(Organization)**

- 여러 개의 AWS 계정을 묶어 하나의 조직으로 관리
    
- 최상위 계정(관리 계정)과 여러 개의 멤버 계정 포함
    

2️⃣ **관리 계정(Management Account)**

- AWS Organizations를 생성하고 관리하는 주 계정
    
- 조직 내 모든 계정의 정책을 설정할 수 있음
    
- 각 계정의 비용 및 리소스를 한 곳에서 확인 가능
    

3️⃣ **멤버 계정(Member Account)**

- 관리 계정에 의해 제어되는 개별 AWS 계정
    
- 조직 내 다른 계정과 리소스를 공유할 수 있음
    

4️⃣ **조직 단위(OU, Organizational Unit)**

- 여러 AWS 계정을 그룹화하는 논리적 단위
    
- OU마다 **다른 정책을 적용**할 수 있음 (예: 보안 정책, 서비스 접근 제한)
    

5️⃣ **서비스 컨트롤 정책(SCP, Service Control Policy)**

- **IAM 정책과 비슷하지만 조직 전체 계정에 강제 적용**되는 정책
    
- 특정 서비스 사용을 제한할 수 있음
    
- 예: "개발팀 계정은 EC2 생성 가능, 하지만 S3는 못 쓰게 하기"

### **🎯 AWS Organizations의 핵심 기능**

| 기능                              | 설명                                 |
| ------------------------------- | ---------------------------------- |
| **계정 중앙 관리**                    | 여러 개의 AWS 계정을 하나의 조직으로 묶어 관리 가능    |
| **통합 청구(Consolidated Billing)** | 모든 멤버 계정의 청구서를 한 곳에서 확인 가능         |
| **서비스 컨트롤 정책(SCP)**             | 조직 내 계정이 AWS 서비스를 어떻게 사용할지 제어 가능   |
| **조직 단위(OU) 설정**                | 여러 계정을 그룹화하고 개별 정책을 적용 가능          |
| **IAM과 연동 가능**                  | SCP와 IAM 정책을 함께 사용하여 보안 강화를 할 수 있음 |

### **🛠 AWS Organizations 사용 사례**

🔹 **여러 계정을 관리하는 기업**  
→ 개발팀, 운영팀, 보안팀 등의 AWS 계정을 따로 만들고, 조직 내에서 중앙 관리

🔹 **비용 절감 & 예산 관리**  
→ 모든 계정의 청구서를 한 곳에서 통합 관리 (통합 청구)

🔹 **보안 강화**  
→ 중요한 리소스가 있는 계정에 **SCP를 적용**하여 불필요한 서비스 접근 차단

🔹 **AWS 환경 표준화**  
→ 개발팀 계정에서는 특정 서비스만 사용하도록 제한 가능
